## 三、测试执行方式

### 3.1 自动化测试范围（60-80%）

#### ✅ 适合完全自动化

1. **API 接口测试**
   - 所有 CRUD 接口的请求/响应格式验证
   - 权限校验（有效 Token/无效 Token/超时 Token）
   - 参数边界值验证（长度/范围/格式）
   - 错误码验证（201/400/401/403/404/409）

2. **数据库测试**
   - 多租户隔离验证：`tenant_id` 查询条件正确
   - 软删除验证：`delete_at IS NULL` 出现在所有查询
   - 数据一致性：操作前后 DB 数据验证
   - 全局共用表验证：`rule_chain` 等无 tenant_id

3. **业务逻辑验证**
   - 费率计算：峰谷平尖正确分段
   - 订单号生成：唯一性 + 格式正确
   - 状态流转：状态码 + 时间戳完整
   - 级联删除：父实体软删后子实体也应软删

4. **权限校验**
   - 权限码映射正确（`service:resource:action`）
   - 超级管理员拥有全部权限
   - 租户管理员仅有本租户操作权限
   - 普通员工仅有分配给该员工的操作权限

#### ⚠️ 半自动化（人工 + 脚本）

1. **UI 交互**
   - 表单验证（前端拦截）：空字段/格式错误
   - 分页加载：大数据量翻页无卡顿
   - 筛选/搜索：结果准确，性能 < 500ms
   - 导出 Excel：格式正确，无乱码

2. **实时推送（SignalR）**
   - WebSocket 连接建立/断线重连
   - 数据推送延迟：< 2s
   - 并发连接数：1000+

3. **地图交互（GIS）**
   - 地图标记加载正确
   - 点击标记弹窗显示
   - 地理位置搜索

#### ❌ 纯手工测试（20-40%）

1. **用户体验（UX）**
   - 页面美观度、交互流畅度
   - 错误提示文案清晰性
   - 界面响应式设计验证（移动端）

2. **第三方集成**
   - OAuth 登录（微信/支付宝）
   - 邮件发送验证（收到激活链接）
   - 短信发送验证（验证码收到）

3. **物理设备交互**
   - 充电桩硬件通信（模拟器验证）
   - MQTT 数据上报
   - 断电/拔枪事件模拟

4. **业务规则验证**
   - 充电时段跨越费率变更
   - 节假日费率覆盖
   - 优惠券叠加规则
