## ⚡ 快速问题排查

### 问题：测试打不开登录页面

**原因**：系统地址不可达  
**解决**：
1. 检查系统是否已启动：`curl http://localhost:3000`
2. 检查防火墙设置
3. 检查网络连接

### 问题：登录失败

**原因**：账户不存在或密码错误  
**解决**：
1. 确认账户：在数据库中查询 `admin@example.com`
2. 重置密码：运行数据库脚本
3. 检查租户配置

### 问题：页面加载很慢

**原因**：系统性能问题  
**解决**：
1. 检查数据库查询日志
2. 检查 API 响应时间
3. 检查网络延迟
4. 重启相关服务

### 问题：测试中途断连

**原因**：网络或服务故障  
**解决**：
1. 检查网络连接
2. 检查服务可用性
3. 检查日志文件
4. 重新启动测试

### 问题：报告无法生成

**原因**：权限或磁盘问题  
**解决**：
1. 检查文件夹权限：`test-results/` 必须可写
2. 检查磁盘空间
3. 检查浏览器日志
4. 手动创建报告目录

## 📞 技术支持

### 常见问题

**Q：三个场景必须按顺序执行吗？**  
A：不必须。但建议先做场景 14（基础），再做 15/16。

**Q：可以跳过某个步骤吗？**  
A：不建议。每个步骤都验证关键功能。如有困难，可反馈调整。

**Q：测试失败了怎么办？**  
A：记录错误信息，生成报告，提交 Issue。

**Q：能修改测试场景吗？**  
A：可以，修改 `scenarios/*.spec.js` 文件，然后重新运行。

**Q：报告在哪里？**  
A：在 `test-results/` 目录，运行 `npm run report` 查看。

- **Bug 反馈**：提交 GitHub Issue（包含截图和错误日志）
- **功能建议**：讨论版或 Pull Request
- **性能问题**：提交性能数据和系统日志
- **其他问题**：联系质量保证团队
