## 🌟 技术特性

### 半自动化方式

每个场景脚本都遵循 **半自动化** 模式：

```
系统高亮元素 → 显示提示框 → 等待用户操作 → 自动截图 → 生成报告
```

**完整工作流示例**（用户管理场景）：

1. **系统引导**
```
┌─────────────────────────┐
│  🙋 用户管理场景开始     │
│  本场景将引导您完成：    │
│  1. 创建新用户          │
│  2. 编辑用户信息        │
│  3. 分配用户角色        │
│  4. 删除测试用户        │
└─────────────────────────┘
```

2. **元素高亮**
```
"新建用户" 按钮 ← 红色边框高亮 + 自动滚动到视图
```

3. **用户确认**
```
┌──────────────────┐
│ ✅ 确认 (Enter)   │
└──────────────────┘
```

4. **自动截图** → 保存到 `test-results/screenshots/`

5. **生成报告** → JSON 格式，包含步骤、截图、元数据

## 🚀 使用指南

### 快速开始

```bash
# 安装依赖
cd tests/manual-test-helpers/playwright-semi-auto
npm install
npx playwright install

# 运行任意场景
npm run scenario:user      # 用户管理
npm run scenario:tenant    # 租户管理
npm run scenario:charging  # 充电工作流
npm run scenario:device    # 设备管理
npm run scenario:workorder # 工单管理
npm run scenario:station   # 站点管理
npm run scenario:permission # 权限管理

# 运行所有场景
npm run test

# 查看测试报告
npm run report
```

### 场景运行流程

1. **浏览器自动打开** - 导航到系统
2. **跟随页面提示** - 按照高亮和提示完成操作
3. **按回车继续** - 继续下一个步骤
4. **完成后查看报告** - `test-results/` 目录

### 典型测试时间

- 单个场景：10-20 分钟（包含用户操作）
- 全部 13 个场景：2-3 小时

## 🔧 故障排除

### Q: 浏览器无法打开？
**A:** 检查 `playwright.config.js` 的 `headless: false` 设置

### Q: 超时了？
**A:** 调整 `timeout` 参数或增加 `waitForUserAction(ms)` 的时间

### Q: 截图保存位置？
**A:** `test-results/screenshots/` 目录（时间戳命名）

### Q: 如何查看 API 监控数据？
**A:** 查看报告 JSON 中的 `apiRequests` 数组
