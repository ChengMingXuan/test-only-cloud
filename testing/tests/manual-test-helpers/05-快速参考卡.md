## 📟 常用命令速查表

### 油猴脚本操作

```bash
# 1. 打开浏览器，访问系统页面
https://localhost:5000/

# 2. 点击浏览器右上角 Tampermonkey 图标 🔧

# 3. 在菜单中选择对应脚本
# 例如：user-create, tenant-create, device-register 等

# 4. 脚本自动执行（5-15 秒）

# 5. 打开浏览器控制台查看输出
# F12 → Console 标签页 → 查看脚本执行结果
```

### Playwright 命令

```bash
# 进入项目目录
cd tests/playwright-scenarios

# 安装依赖（仅首次）
npm install

# 列出所有可用场景
npm run list

# 运行单个场景
npm run scenario:01          # 用户管理
npm run scenario:energy-core # 能源核心基础设施
npm run scenario:energy-all  # 全部能源场景（14-16）

# 运行多个场景
npm run scenarios:basic      # 场景 01-06
npm run scenarios:advanced   # 场景 07-12
npm run scenarios:all        # 场景 01-16

# 高级选项
npm run scenario:01 -- --debug      # 调试模式
npm run scenario:01 -- --headed     # 显示浏览器（默认）
npm run scenario:01 -- --headless   # 不显示浏览器

# 查看报告
npx playwright show-report
```

### 系统管理命令

```bash
# Docker 操作
docker-compose up -d          # 启动系统
docker-compose restart        # 重启系统
docker-compose down           # 停止系统
docker-compose logs           # 查看日志

# 检查系统是否运行
curl http://localhost:5000/   # 应该返回 HTML

# 清理测试数据（参考清理脚本）
./scripts/cleanup-test-data.sh
```

## 🔍 故障快速排查

### 问题速查（最常见的 5 个问题）

```
❌ 问题：浏览器启动失败
→ 解决：npm install && npx playwright install chromium

❌ 问题：连接到系统失败
→ 解决：docker-compose up -d && curl http://localhost:5000/

❌ 问题：油猴脚本不显示
→ 解决：检查 @match 规则，应该是 https://localhost:5000/*

❌ 问题：Playwright 超时
→ 解决：重启系统，增加超时时间，检查网络

❌ 问题：API 返回 401
→ 解决：重新登录，检查权限和会话
```

详细解决方案参考：[故障排查指南](./04-故障排查指南.md)
