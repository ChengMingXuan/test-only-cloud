## 📋 按问题类型分类

### 🔧 安装和配置问题

#### Q1: Playwright 安装失败

**现象**：
```
Error: ENOENT: no such file or directory,
open 'C:\Users\...\AppData\Local\ms-playwright\chromium-xxx\chrome.exe'
```

**原因分析**：
- Playwright 浏览器文件未完整下载
- 磁盘空间不足（需要 > 500MB）
- 网络连接中断

**解决步骤**：

```bash
# 第 1 步：清除已有的浏览器缓存
rm -rf ~/.cache/ms-playwright/  # macOS/Linux
rmdir /s %APPDATA%\Local\ms-playwright\  # Windows

# 第 2 步：检查磁盘空间
df -h  # Linux/macOS
dir C:\  # Windows

# 第 3 步：重新安装 Playwright
npx playwright install

# 第 4 步：验证安装
npx playwright test --version
```

**预期结果**：
```
✅ Chromium 121.0
✅ Firefox 121.0
✅ WebKit 17.4
```

#### Q2: 油猴脚本不显示在菜单中

**现象**：
安装了脚本，但点击 Tampermonkey 菜单后看不到脚本

**原因分析**：
- 脚本的 @match 规则不匹配当前页面
- 脚本执行被禁用
- 浏览器缓存问题

**解决步骤**：

```javascript
// 检查 @match 规则是否正确

// ❌ 错误的例子：
// @match https://localhost:5000/user/create
// 这只匹配这一个 URL，太严格了

// ✅ 正确的例子：
// @match https://localhost:5000/*
// 这匹配所有 localhost:5000 下的页面

// 你的系统地址：https://localhost:5000 或 http://192.168.1.100:8080
// 对应的 @match 应该是：https://localhost:5000/* 或 http://192.168.1.100:8080/*
```

**解决方法**：

```
1. 在 Tampermonkey 菜单中点击"管理脚本"
2. 找到你的脚本，点击编辑
3. 修改 @match 规则为：https://[你的系统地址]/*
4. 点击保存（Ctrl+S）
5. 回到系统页面，刷新
6. 重新尝试
```

#### Q3: npm 命令找不到

**现象**：
```
Error: command not found: npm
或
'npm' is not recognized as an internal or external command
```

**原因分析**：
- Node.js 没有安装
- npm 没有添加到系统 PATH

**解决步骤**：

```bash
# 第 1 步：检查 Node.js 是否安装
node --version
npm --version

# 如果两个命令都找不到，说明 Node.js 未安装

# 第 2 步：安装 Node.js
# 访问 https://nodejs.org/ 下载 LTS 版本
# 安装（一路 Next）

# 第 3 步：重新打开终端（让新的 PATH 生效）

# 第 4 步：验证安装
node --version  # 应该显示 v18.x 或更高
npm --version   # 应该显示 9.x 或更高
```

### 🚀 运行和执行问题

#### Q4: 浏览器启动超时

**现象**：
```
Error: browserType.launch() timed out after 30000ms
```

**原因分析**：
- 系统资源不足（内存、CPU）
- 浏览器启动很慢
- 防火墙阻止

**解决步骤**：

```bash
# 方法 1：关闭其他应用，释放内存
# 在 Windows 任务管理器中关闭不必要的应用
# 在 macOS Activity Monitor 中关闭不必要的应用
# 在 Linux 使用 free -h 查看内存

# 方法 2：增加超时时间
# 编辑 playwright.config.js
timeout: 60000  // 从 30000 改为 60000（60 秒）

# 方法 3：重新安装浏览器
npx playwright install chromium
```

#### Q5: 连接到系统失败（ERR_CONNECTION_REFUSED）

**现象**：
```
Error: net::ERR_CONNECTION_REFUSED at http://localhost:5000
```

**原因分析**：
- 测试系统未启动
- 系统地址配置错误
- 端口被占用

**解决步骤**：

```bash
# 第 1 步：检查系统是否运行
curl http://localhost:5000/

# 如果返回 "Connection refused"，说明系统未启动

# 第 2 步：启动系统
# 如果使用 Docker:
docker-compose up -d

# 如果使用本地运行：
# 在 Visual Studio 中按 F5 启动

# 第 3 步：等待系统完全启动（1-2 分钟）

# 第 4 步：再次检查连接
curl http://localhost:5000/

# 应该返回 HTML 内容或状态信息
```

#### Q6: Playwright 场景运行中途卡住

**现象**：
脚本运行到某个步骤后，长时间无响应，且出现"等待用户确认"的提示

**原因分析**：
- 页面加载缓慢，脚本等待超时
- 网络连接不稳定
- 系统后端处理缓慢

**解决步骤**：

```bash
# 方法 1：增加页面加载超时
# 编辑具体的 .spec.js 文件，修改 waitForNavigation 超时时间
await page.waitForNavigation({ timeout: 60000 });  // 改为 60 秒

# 方法 2：查看网络延迟
# 打开浏览器 F12 → Network 标签页
# 观察 API 请求响应时间
# 如果 > 30 秒，说明后端有问题，需要联系开发人员

# 方法 3：重启系统
docker-compose restart

# 方法 4：跳过此步骤（临时方案）
# 在没有标记为 CRITICAL 的步骤中，可以按 ❌ 标记为失败
# 然后继续下一步
```

#### Q7: API 响应返回 401（Unauthorized）

**现象**：
```
API Response: 401 Unauthorized
Error: Authentication failed
```

**原因分析**：
- 会话超时（登录被踢出）
- 身份验证令牌失效
- 权限不足

**解决步骤**：

```bash
# 第 1 步：手动登出并重新登录
# 在浏览器中访问登出按钮或刷新页面

# 第 2 步：清除浏览器缓存和 Cookies
# F12 → Application/Storage → Cookies
# 删除所有 cookies

# 第 3 步：重新开始测试
npm run scenario:01

# 如果仍然失败，说明是权限问题
# 第 4 步：检查当前用户的权限
# 在 Permission 管理页面查看用户权限是否包含所需的权限码
```

#### Q8: API 响应返回 400（Bad Request）

**现象**：
```
API Response: 400 Bad Request
Error: Validation failed - "email" field is required
```

**原因分析**：
- 生成的数据不符合 API 要求（空值、格式错误等）
- 后端验证规则改变
- 必填字段缺失

**解决步骤**：

```bash
# 第 1 步：查看完整的错误信息
# 打开浏览器 F12 → Network → 找到红色的 POST/PUT 请求
# 点击 → Response 标签页 → 查看具体错误信息

# 典型的错误响应：
{
  "code": "VALIDATION_ERROR",
  "message": "Validation failed",
  "details": [
    {
      "field": "email",
      "message": "Invalid email format"
    }
  ]
}

# 第 2 步：根据错误修改脚本中的数据生成逻辑
// 例如：邮箱格式错误
// 修改前：const email = `${timestamp}@test}`;  // 缺少 .com
// 修改后：const email = `${timestamp}@test.com`;  // 正确

# 第 3 步：重新测试
npm run scenario:01
```

### 🔍 结果验证问题

#### Q9: 测试通过但实际功能有问题

**现象**：
脚本返回"成功"，但页面上看不到预期的变化

**原因分析**：
- 脚本只检查了 API 响应，没有检查实际的 UI 变化
- 页面缓存导致看不到新数据
- 数据库还没有更新

**解决步骤**：

```
1. 手动刷新页面（F5）
   └─ 确保显示的是最新数据

2. 检查浏览器开发工具
   └─ F12 → Elements 找到相关元素
   └─ 查看是否包含你创建的数据

3. 查看浏览器的 Network 标签页
   └─ 查看 API 响应是否真的成功（HTTP 200）
   └─ 查看响应数据是否正确

4. 查看后端日志
   └─ 如果可以访问后端日志，查看操作是否真的执行了
   └─ 可能在某个中间件中被拒绝了
```

#### Q10: 报告显示失败，但不知道失败原因

**现象**：
```
❌ 步骤 3：编辑用户 - FAILED
错误信息：Timeout waiting for user confirmation
```

**原因分析**：
这通常意味着脚本等待了很长时间（通常是 API 响应慢或页面未加载）

**解决步骤**：

```
1. 查看测试报告中的截图
   └─ test-results/report.html → 找到失败的步骤 → 点击查看截图
   └─ 截图会显示当时页面的状态（成功？错误提示？）

2. 查看网络性能指标
   └─ 报告中的"Network"部分
   └─ 观察 API 响应时间（应 < 1000ms）
   └─ 如果 > 3000ms，说明后端很慢

3. 打开浏览器调试（Debug Mode）
   └─ npm run scenario:01 -- --debug
   └─ 逐步执行，在每个步骤观察浏览器和网络状态

4. 查看 Trace 记录
   └─ 如果启用了 --trace on
   └─ 打开 trace 文件：npx playwright show-trace test-results/trace.zip
   └─ 可以看到每一步的详细执行过程
```

#### Q11: 油猴脚本控制台无输出

**现象**：
运行脚本但浏览器控制台什么都没有显示

**原因分析**：
- 脚本没有正确执行
- 控制台没有打开或被最小化
- 脚本有错误导致提前退出

**解决步骤**：

```
1. 打开浏览器控制台
   └─ F12 → Console 标签页
   └─ 如果显示"Console was cleared"，说明脚本确实运行过

2. 查看是否有错误信息
   └─ 红色行提示 JavaScript 错误
   └─ 记下错误信息，可能是元素找不到或 API 改变了

3. 逐个运行脚本，排序问题
   └─ 对脚本代码添加调试日志
   └─ 例如在脚本开始加入：console.log('脚本开始执行');

4. 检查脚本 @match 规则
   └─ 如前所述，规则不匹配会导致脚本不运行
```

## 🆘 常见错误信息解释

### Playwright 错误

| 错误信息 | 原因 | 解决方案 |
|----------|------|---------|
| `Timeout` | 等待超时 | 增大超时时间或检查网络 |
| `Connection refused` | 系统未启动 | 启动系统 docker-compose up |
| `Navigation Timeout` | 页面加载太慢 | 等待或重启系统 |
| `Element not found` | 页面元素改变 | 更新脚本选择器 |
| `Failed to load resource` | 网络错误 | 检查网络连接 |

### API 错误

| HTTP 状态码 | 含义 | 常见原因 |
|-----------|------|---------|
| 400 | Bad Request | 数据格式错误 |
| 401 | Unauthorized | 未登录或权限不足 |
| 403 | Forbidden | 禁止访问 |
| 404 | Not Found | API 端点改变 |
| 500 | Server Error | 后端异常 |
| 503 | Service Unavailable | 服务维护中 |

### 快速问题排查清单

在询问他人之前，请检查以下几点：

- [ ] 系统是否正常运行？（访问页面能看到内容）

## 🎯 最常见的 3 个问题（快速答案）

### 问题 1：我想快速验证某个功能（< 5 分钟）
**答案**：使用油猴脚本
- 选择对应功能的脚本
- 点击 Tampermonkey 菜单运行
- 查看控制台输出结果

### 问题 2：我想完整测试一个新模块
**答案**：使用 Playwright（如果有对应场景）
- npm run scenario:XX（查看编号）
- 按照系统提示逐步进行测试
- 查看生成的 HTML 报告

### 问题 3：我的脚本/场景失败了，怎么办？
**答案**：按以下顺序操作
1. 重新运行一次（可能是临时网络问题）
2. 检查系统是否正常（访问页面能看到内容）
3. 查看错误详情（控制台或报告中的截图）
4. 根据错误原因修复或报告问题
