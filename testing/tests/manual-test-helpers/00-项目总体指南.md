## 🎯 项目概述

### 使命
为 AIOPS 项目提供**完整、灵活、高效**的手动和半自动化测试工具，支持从快速冒烟测试到深度业务流程验证的全面覆盖。

### 核心特点
- 🔹 **多层次覆盖**：从快速油猴脚本到深度 Playwright 测试
- 🔹 **灵活选择**：根据需求选择合适的测试方式
- 🔹 **人机结合**：保留人工判断空间，同时自动化重复工作
- 🔹 **全无脚本**：用户交互数据驱动，无需编写测试代码
- 🔹 **完整报告**：详细的截图、日志、性能指标

## 📚 三种测试方式详解

### 1️⃣ Tampermonkey (油猴脚本) - 快速验证工具

**何时使用**：
- ✅ 快速验证单个功能是否正常
- ✅ 进行冒烟测试
- ✅ 验证 UI 交互和表单操作
- ✅ 实时反馈和调试

**不适合的场景**：
- ❌ 需要完整的业务流程验证
- ❌ 需要复杂的等待和时序控制
- ❌ 需要详细的测试报告
- ❌ 需要 API 监控和性能指标

**特点**：
- 🟢 轻量级，运行速度快
- 🟢 代码直接运行在浏览器中
- 🟢 可以实时注入数据和 UI 元素
- 🟢 适合快速迭代和调试

**文档**：
👉 详见 [油猴脚本完整指南](../tampermonkey/TAMPERMONKEY_COMPLETE_GUIDE.md)

### 2️⃣ Playwright (半自动化) - 综合测试框架

**何时使用**：
- ✅ 需要完整的业务流程验证
- ✅ 需要用户交互的参与和确认
- ✅ 需要详细的测试报告和截图
- ✅ 需要监控 API 请求和响应

**不适合的场景**：
- ❌ 需要极速反馈（< 5 秒）
- ❌ 不适合频繁变动的快速迭代
- ❌ 页面异常复杂时配置会很繁琐

**特点**：
- 🟢 完整的测试框架，支持多种浏览器
- 🟢 详细的报告、截图、日志
- 🟢 支持 API 监控和数据验证
- 🟢 可扩展的架构

**文档**：
👉 详见 [Playwright 完整指南](./PLAYWRIGHT_COMPLETE_GUIDE.md)

### 3️⃣ 完全自动化测试 - 高级选项

**说明**：项目提供的是**半自动化**测试（系统引导 + 用户确认），这保留了人工判断的空间。

如果需要**完全自动化**（零人工干预），有两个选项：
1. **基于 Playwright 扩展**：直接编写断言替代用户确认
2. **使用 Jest / Vitest**：编写单元测试和集成测试

## 🚀 快速开始（5 分钟）

### 前置要求
```bash
# 1. 安装 Node.js (≥ 18.0.0)
node --version   # 需要 v18.0.0+

# 2. Tampermonkey 脚本
# - 浏览器已安装 Tampermonkey 扩展
# - 访问 https://greasyfork.org/ 导入脚本

# 3. Playwright 环境
npm install && npx playwright install
```

### 运行测试

#### 🟠 快速验证一个功能（5-15 分钟）

```bash
# 使用 Tampermonkey 油猴脚本
# 打开浏览器 → 访问系统 → 点击脚本菜单 → 选择功能
# 例如："智能表单填充" → 自动填充并提交

# 耗时：5-15 分钟
# 输出：直接反馈 + 控制台日志
```

#### 🟡 系统地验证一个完整业务流程（30-60 分钟）

```bash
cd tests/manual-test-helpers/playwright-semi-auto

# 运行单个测试场景
npm run scenario:user        # 用户管理流程
npm run scenario:charging    # 充电工作流
npm run scenario:e2e         # 端到端完整流程

# 耗时：30-60 分钟
# 输出：详细报告 + 截图 + HTML 报表
```

#### 🔴 全方位深度验证（2+ 小时）

```bash
cd tests/manual-test-helpers/playwright-semi-auto

# 运行完整的测试套件
npm run test:all            # 运行全部 16 个场景

# 或指定能源专项
npm run scenario:energy-all  # 运行全部 11 个能源微服务

# 耗时：2+ 小时
# 输出：完整报告 + API 监控日志 + 性能指标 + HTML 报表
```
