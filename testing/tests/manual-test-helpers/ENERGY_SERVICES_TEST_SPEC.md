## 📋 执行摘要

本文档定义了 AIOPS 项目中 **11 个能源微服务** 的完整测试规范，包括：

- **3 个能源核心基础设施服务**：MicroGrid、Orchestrator、PVESSC、VPP
- **3 个能源交易和商业服务**：ElecTrade、CarbonTrade、EnergyEff
- **4 个能源运维和控制服务**：DeviceOps、DemandResp、MultiEnergy、SafeControl

**覆盖形式**：3 个 Playwright 测试场景（scenario:14-16）  
**总测试步骤**：18 个（每个场景 6 个步骤）  
**预期耗时**：120-180 分钟（三个场景全部运行）

## 📊 详细测试清单

### 测试清单 - Scenario 14：能源基础设施

#### 步骤 1：微网配置和拓扑验证

**前置条件**：
- ✅ 用户已登录为能源管理员
- ✅ MicroGrid 服务正常运行
- ✅ 基础设备数据已加载

**测试步骤**：
1. 导航到"微网管理"页面
2. 创建一个新微网（名称：Auto_MicroGrid_001）
3. 配置微网参数：
   - 位置：测试园区
   - 额定功率：500kW
   - 储能容量：200kWh
   - 接入方式：并网
4. 添加微网节点（至少 3 个）：
   - 光伏站点 x1
   - 储能系统 x1
   - 充电桩 x1
5. 验证拓扑关系图显示正确
6. 保存配置并返回列表

**预期结果**：
- ✅ 微网创建成功，ID 生成正确
- ✅ 参数保存无误
- ✅ 拓扑图显示所有节点和连接
- ✅ 节点间通信延迟 < 100ms

**验证方法**：
1. 检查数据库 microgrids 表中是否有新记录
2. 查看 API 响应时间（应 < 500ms）
3. 观察页面拓扑图的完整性
4. 检查网络监控中的节点通信状态

**失败时的调查**：
- [ ] 微网创建失败 → 检查 MicroGrid 服务日志
- [ ] 参数保存不完整 → 检查输入验证规则
- [ ] 拓扑图显示不正确 → 检查拓扑算法实现
- [ ] 通信延迟过高 → 检查网络配置

---

#### 步骤 2：发电计划和调度验证

**前置条件**：
- ✅ 微网已创建（来自步骤 1）
- ✅ Orchestrator 服务正常运行
- ✅ 天气预报数据已加载

**测试步骤**：
1. 导航到"发电计划"页面
2. 创建发电计划：
   - 起始日期：明天
   - 计划周期：24 小时
   - 预期发电量：1000 kWh
3. 系统自动根据天气预报生成分时发电曲线
4. 验证曲线合理性（高峰在正午）
5. 手动调整不合理的时段
6. 下达计划给相关发电单位

**预期结果**：
- ✅ 计划创建成功
- ✅ 曲线生成合理，符合物理规律
- ✅ 调整能实时生效
- ✅ 下达成功率 100%

**验证方法**：
1. 检查计划是否在数据库中保存
2. 查看曲线是否符合天气预报
3. 检查调度指令是否送达各单位
4. 验证发电单位是否收到通知

## 🎯 关键指标和成功标准

### 功能指标

| 指标 | 目标 | 评分方式 |
|------|------|---------|
| **场景通过率** | ≥ 95% | 通过步骤数 / 总步骤数 × 100% |
| **API 响应时间** | < 500ms | 平均响应时间 |
| **可用性** | ≥ 99% | (总请求数 - 失败数) / 总请求数 |
| **数据准确率** | 100% | 无数据一致性问题的步骤数 / 总步骤数 |

### 业务指标

| 指标 | 含义 | 通过标准 |
|------|------|---------|
| **微网通信延迟** | 节点间信息传递时间 | < 100ms |
| **调度响应时间** | 从指令下达到执行的时间 | < 60 秒 |
| **市场成交率** | 提交的报价有多少被成交 | ≥ 80% |
| **交易结算准确率** | 结算金额无误的交易占比 | 100% |

## 🚀 运行命令

### 运行单个场景

```bash
cd tests/playwright-scenarios

# 能源基础设施
npm run scenario:14

# 能源交易
npm run scenario:15

# 能源运维
npm run scenario:16
```

### 运行全部能源场景

```bash
# 推荐：一次运行所有 3 个能源场景
npm run scenario:energy-all

# 或分别运行（如果其中一个失败，可先排查再继续）
npm run scenario:14
npm run scenario:15
npm run scenario:16
```

### 查看报告

```bash
# 生成 HTML 报告
npx playwright show-report

# 查看最新报告
open test-results/report.html  # macOS
xdg-open test-results/report.html  # Linux
start test-results/report.html  # Windows
```

## 📞 故障排查

### 常见问题

**Q1：Scenario 14 在"微网配置"步骤失败**
- 原因可能：MicroGrid 服务未启动或异常
- 解决：检查后端日志，确认 MicroGrid 服务健康状态
- 命令：`docker-compose logs jgsy-energycore-microgrid`

**Q2：Scenario 15 交易成交率低**
- 原因可能：市场引擎算法问题或报价参数不合理
- 解决：检查市场引擎日志，调整报价策略
- 命令：`docker-compose logs jgsy-energyservices-electrade`

**Q3：Scenario 16 故障检测延迟长**
- 原因可能：监控告警规则未生效或告警规则过松散
- 解决：检查告警规则配置，确保阈值设置合理
- 命令：查看告警规则管理页面
