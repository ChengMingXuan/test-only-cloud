# AIOPS 油猴脚本套件 - 5分钟快速上手

## 📦 8 个脚本，1 种方式安装

### 安装步骤（3 步）

**第 1 步**：打开 Tampermonkey 仪表板
```
Chrome/Edge：点击工具栏 Tampermonkey 图标 → "管理脚本"
Firefox：点击工具栏 Tampermonkey 图标 → "管理脚本"
或者访问：chrome-extension://[ID]/options.html
```

**第 2 步**：逐个创建脚本
```
1. 点击"创建新脚本"
2. 删除默认内容
3. 复制下列任一脚本的全部内容
4. 粘贴到编辑器
5. Ctrl+S 保存
6. 脚本自动启用
```

**第 3 步**：验证安装成功
```
- 访问 http://localhost:8000 或 http://localhost:端口
- 按快捷键（如 Alt+F）
- 看到浮窗表示安装成功 ✅
```

## 📊 8 个脚本详细说明

### 1️⃣ 表单自动填充 Pro 👉 **aiops-form-autofill-pro.user.js**

**作用**：快速填充后台表单，自动生成测试数据

**快捷键**：
```
Alt+F  → 自动填充（智能检测页面类型）
Alt+C  → 清空所有字段
Alt+R  → 随机数据（每个字段不同随机值）
Alt+S  → 提交表单（自动点击保存按钮）
```

**支持的表单**（18 种）：
- 平台：用户、租户、角色、权限、菜单
- 业务：充电订单、充电桩、设备、工单、站点
- 高级：结算、规则、告警、报告、账户

**例子**：
```
场景：创建新用户
1. 访问 /admin/users/new
2. 按 Alt+F → 表单自动填充
3. 按 Alt+S → 自动保存

耗时：5 秒（vs 手工 2 分钟）
```

---

### 2️⃣ API 调试器 👉 **aiops-api-debugger.user.js**

**作用**：实时看到所有 API 请求和响应

**快捷键**：
```
Alt+A  → 显示/隐藏 API 监控面板
```

**能看到什么**：
```
GET /api/users → 200 ✅ (120ms)
POST /api/orders → 201 ✅ (850ms)  
GET /api/devices → 500 ❌ (2500ms)  ← 这个请求太慢！

性能分类：
✅ 绿色（< 500ms）：快
⚠️ 黄色（500-1000ms）：慢
❌ 红色（> 1000ms）：非常慢
```

**实用场景**：
```
1. 页面卡顿 → 打开 API 调试器 → 看哪个请求慢
2. 数据没更新 → 看是否有 4xx/5xx 错误
3. 接口优化 → 监控优化前后的速度对比
```

### 3️⃣ 数据操作工具 👉 **aiops-data-operator.user.js**

**作用**：快速创建、导入导出测试数据

**快捷键**：
```
Alt+D  → 显示/隐藏数据操作面板
```

**功能**：
```
📝 批量创建
  | 输入数字（7）→ 批量创建 7 个用户
  | "快速创建 5 个设备"（一键）
  | "快速创建 10 个工单"（一键）

📤 导入导出
  | "导出表格"（当前页表格 → CSV）
  | "导入 Excel"（选择文件 → 上传）
  | "导出选中行"（选中某些行 → CSV）

🗑️ 清理数据
  | "清理测试数据"（删除 test_* 的数据）
  | "清理缓存"（清除浏览器存储）
```

**例子**：
```
场景：准备 100 个测试订单
1. 按 Alt+D 打开
2. 输入 100 → 点击"批量创建用户"（耗时 30 秒）
3. 不用一个个手工创建！
```

### 4️⃣ 权限测试器 👉 **aiops-permission-tester.user.js**

**作用**：快速切换角色，检查权限是否正确

**快捷键**：
```
Alt+P  → 显示/隐藏权限测试面板
```

**快速切换 4 种角色**：
```
点击卡片即可：
┌─────────────────┐
│ SUPER_ADMIN     │ → 超级管理员（全部权限）
│ Admin           │ → 普通管理员（大部分权限）
│ User            │ → 普通用户（有限权限）
│ Viewer          │ → 查看者（只读）
└─────────────────┘
```

**检查权限**：
```
当前显示 4 个权限复选框：
☑ Create（创建）
☑ Edit（编辑）
☑ Delete（删除）
☐ Export（导出）← 当前角色无此权限

点击"验证权限"→ 检查是否配置正确
```

**检查访问权限**：
```
点击 6 个路径之一：
[ /admin/users ]
[ /admin/permissions ]
[ /charging/orders ]  
[ /device ]
[ /work-order ]
[ /settlement ]

或者 [ 批量测试所有功能 ]

结果：
✅ 有权 (200)
⚠️ 未认证 (401)
❌ 禁止访问 (403)
```

### 6️⃣ 多租户验证 👉 **aiops-multitenant-verifier.user.js**

**作用**：确保不同租户的数据不会混淆

**快捷键**：
```
Alt+T  → 显示/隐藏多租户验证面板
```

**快速切换 4 个租户**：
```
点击卡片即可：
┌─────────────────────────────┐
│ 全局租户                     │ → 系统租户
│ Tenant-A                     │ → 测试租户 A
│ Tenant-B                     │ → 测试租户 B
│ Tenant-C                     │ → 测试租户 C
└─────────────────────────────┘
```

**自动检查数据隔离**：
```
点击"检查"按钮，自动验证 8 个 API：
/api/users        ✅ 隔离正确
/api/tenants      ✅ 隔离正确
/api/roles        ✅ 隔离正确
/api/permissions  ✅ 隔离正确
/api/devices      ⚠️ 需检查      ← 某个 API 可能有问题
...
```

**高级测试**：
```
点击"检测数据泄露"
  → 在 Tenant-A 创建数据
  → 切换到 Tenant-B
  → 检查 Tenant-B 是否能看到 Tenant-A 的数据
  结果："✅ 数据隔离正确" 或 "❌ 检测到泄露"

点击"检测权限越界"
  → 普通用户访问 admin API
  → 应该被拒绝（403）
  结果："✅ 权限验证正确" 或 "❌ 权限验证失败"
```

### 7️⃣ 日志聚合 👉 **aiops-log-aggregator.user.js**

```

**例子**：
```
实时日志流：
14:23:45  [API]    GET /api/users [200]          ✅
14:23:46  [API]    POST /api/orders [201]        ✅

14:23:48  [API]    GET /api/devices [500]        ❌
14:23:49  [Info]   Page loaded successfully      ℹ️
```

**统计汇总**（屏幕底部）：
```
总数：248 条    错误：2 条    警告：5 条    API：87 条
```

**导出日志**：
```
点击"导出"→ JSON 文件下载
可用于：
  1. 发送给后端团队
  2. 离线分析

## ❓ 快速解决

| 问题 | 解决方案 |
|------|--------|
| 脚本按键无反应 | Ctrl+Shift+R 硬刷新页面 |
| 表单未填充 | 尝试 Alt+C 清空，再按 Alt+F |
| API 监控看不到请求 | 浏览器刷新，然后执行某个操作 |
| 无法切换租户 | 检查浏览器是否隐私模式 |
| 性能指标为 N/A | 这是正常的，刷新页面再看 |
| 日志太多卡顿 | 点击"清空"删除日志 |
