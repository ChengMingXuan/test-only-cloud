# AIOPS 完整手动测试工具套件

## 📦 核心概念

这是一套**全方位（全覆盖）**的手动测试辅助工具，旨在补充自动化测试的不足（自动化 80% + 手动 20%）。

包含：
- ✅ **8 个油猴脚本**（Tampermonkey）- 浏览器端测试工具
- ✅ **Playwright 半自动脚本**（可选）- 接近自动化的引导式测试
- ✅ **完整文档**（快速上手 + 深度指南）

---

## 🎯 为什么需要这套工具？

### 自动化测试的局限
- ❌ 无法覆盖 100% 的场景
- ❌ 修改页面结构后需要大量调整
- ❌ UI 交互式操作难以测试
- ❌ 多租户、多角色场景复杂

### 手动测试的痛点
- ❌ 重复创建测试数据耗时
- ❌ 不同角色要反复登录
- ❌ API 请求难以跟踪
- ❌ 权限验证需要逐个检查

### 本套件的解决方案
- ✅ **一键自动填表** - 18+ 模块智能识别
- ✅ **实时 API 监控** - 看到每一个请求
- ✅ **快速角色切换** - 无需重新登录
- ✅ **批量数据创建** - 秒级完成
- ✅ **性能实时分析** - 瓶颈一目了然
- ✅ **多租户隔离验证** - 自动化检测

## 🚀 快速开始（3 步）

### 1️⃣ 安装 Tampermonkey
- **Chrome**: [Chrome Web Store - Tampermonkey](https://chrome.google.com/webstore)
- **Firefox**: 搜索"Tampermonkey"安装
- **Edge**: Microsoft Edge 扩展商店

### 2️⃣ 安装 8 个脚本
- 打开 Tampermonkey 仪表板
- 逐个复制脚本内容，创建新脚本
- Ctrl+S 保存（脚本自动启用）

### 3️⃣ 验证安装
- 访问 http://localhost:8000
- 按 **Alt+F** 测试表单填充
- 应该看到浮窗弹出 ✅

## 🔄 典型测试流程

### 场景 1️⃣：完整功能测试（新建工单）

```
1. [Alt+D] 打开数据操作工具
   └─ 创建测试用户、设备等前置数据

2. [Alt+F] 打开表单填充
   └─ 自动填充新建工单表单
   └─ Alt+S 提交

3. [Alt+A] 监控 API
   └─ 看创建工单的请求是否成功
   └─ 记录响应时间

4. [Alt+P] 权限验证
   └─ 切换不同角色
   └─ 验证每个角色的权限

5. [Alt+L] 检查日志
   └─ 确保无错误或警告

6. [Alt+E] 性能分析
   └─ 页面加载时间是否在标准范围
```

### 场景 2️⃣：性能基准测试

```
1. [Alt+E] 打开性能分析器
2. 清空浏览器缓存
3. 硬刷新页面 (Ctrl+Shift+R)
4. 记录关键指标：
   - 页面加载时间
   - TTFB (首字节时间)
   - LCP (最大内容绘制)
   - 性能评分
5. 与历史数据对比
```

### 场景 3️⃣：多租户隔离验证

```
1. [Alt+T] 打开多租户验证
2. 点击"检测数据泄露"
   └─ 创建数据在 Tenant-A → 切换到 Tenant-B
   └─ 结果："✅ 通过" 或 "❌ 失败"
3. 点击"检测权限越界"
   └─ 普通用户访问 admin API
   └─ 应该返回 403 Forbidden
```

## 🔧 高级配置

### 修改快捷键

找到脚本中的快捷键定义（通常在最后）：

```javascript
// 示例：修改 Alt+F 为 Ctrl+Shift+F
if (e.altKey && (e.key === 'f' || e.key === 'F')) {
  // 改为：
  if (e.ctrlKey && e.shiftKey && (e.key === 'f' || e.key === 'F')) {
    // ... rest of code
  }
}
```

### 自定义测试数据

编辑脚本中的 `CONFIG.TEST_DATA` 对象：

```javascript
CONFIG.TEST_DATA.user = {
  name: '你的测试用户名',
  email: 'your.email@domain.com',
  phone: '13800000000',
  // ... 其他字段
};
```

### 添加新模块支持

1. 在 `PageDetector.detect()` 中添加页面识别规则
2. 在 `FormFillers` 中添加对应的 `fill[ModuleName]Form()` 方法
3. 在 `CONFIG.TEST_DATA` 中添加测试数据
