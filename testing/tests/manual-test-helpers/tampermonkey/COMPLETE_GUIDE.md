## 快速开始

### 前置条件

1. **安装浏览器扩展**：Tampermonkey（适用于 Chrome、Firefox、Edge 等）
   - Chrome：[Tampermonkey - Chrome Web Store](https://chrome.google.com/webstore)
   - Firefox：搜索"Tampermonkey"并安装

2. **验证 Tampermonkey 已启用**：
   - 点击浏览器工具栏的 Tampermonkey 图标
   - 确保扩展状态为"已启用"

### 安装脚本

1. **一键安装**（推荐）：在 Tampermonkey 控制板点击"创建新脚本"，复制以下任何脚本内容并运行

2. **逐个安装**：
   - 打开 Tampermonkey 仪表板
   - 点击"创建新脚本"
   - 将下列 .user.js 文件内容复制粘贴
   - 点击 ✅ 保存（Ctrl+S）
   - 脚本将自动启用

## 详细使用指南

### 1️⃣ 表单自动填充 Pro (`aiops-form-autofill-pro.user.js`)

**功能概述**：
- 支持 18+ AIOPS 模块类型的智能表单填充
- 自动检测页面类型并应用模块特定的填充规则
- 支持 React 受控组件、Vue、Angular 等框架

**使用方法**：

#### 🚀 智能填充（Alt+F）
```
在任何表单页面，按 Alt+F 自动填充：
- 自动检测页面类型（用户、租户、设备等）
- 使用预配置的测试数据填充相应字段
- 智能处理各种输入框、下拉菜单、文本域
```

#### 🗑️ 清空表单（Alt+C）
```
清除当前页面所有表单字段
- 所有 input 字段恢复为空
- 下拉菜单恢复默认值
- 文本域清空
```

#### 🎲 随机数据（Alt+R）
```
使用随机数据填充表单
```

**右键菜单支持**：
```
在输入框上右键 → 选择：
- 自动填充（当前模块）
```

### 2️⃣ API 调试器 (`aiops-api-debugger.user.js`)

**功能概述**：
- 实时拦截和显示所有 API 请求和响应
- 性能分析：延迟、文件大小、成功率统计
- 色码标记：绿色(快)、黄色(慢)、红色(错误)

**使用方法**：

#### 打开/关闭（Alt+A）
```
按 Alt+A 显示或隐藏 API 监控面板
面板位置：屏幕右侧
```

#### 查看请求日志
```
实时显示：
- 请求方法 (GET/POST/PUT/DELETE)
- 请求 URL
- 响应状态码
- 响应时间（毫秒）
- 响应大小（KiB）
```

#### 性能分类
```
✅ 快（绿色）：< 500ms
⚠️ 中等（黄色）：500-1000ms
❌ 慢（红色）：> 1000ms
```

#### 统计信息
```
显示在面板底部：
- 总请求数
- 错误请求数
- 平均响应时间
- 最大响应时间
```

#### 控制按钮
```
- 🔄 刷新：清除当前日志并重新监控
- 🗑️ 清空：删除所有日志记录
- ❌ 关闭：隐藏面板
```

**性能优化建议**：
```
基础标准：
- TTFB < 500ms (第一字节时间)
- API 响应 < 1000ms
- 完整加载 < 3000ms

如果看到红色请求，检查：
1. 网络连接
2. 服务器负载
3. 请求 payload 大小
4. API 端点性能
```

### 5️⃣ 性能分析器 (`aiops-performance-analyzer.user.js`)

**功能概述**：
- 关键 Web 指标（Core Web Vitals）
- 阶段性能分析（DNS、TCP、DOM 等）
- 资源加载和内存占用监控

**使用方法**：

#### 打开/关闭（Alt+E）
```
按 Alt+E 显示或隐藏性能分析面板
面板位置：屏幕右上角
```

#### 📊 关键指标

```
页面加载时间：总时间
TTFB（首字节时间）：服务器响应速度
LCP（最大内容绘制）：用户感知到内容
FID（首次输入延迟）：交互响应性
CLS（累积布局偏移）：视觉稳定性

标准参考：
✅ 优秀：
  - 页面加载 < 1000ms
  - TTFB < 500ms
  - LCP < 2500ms

⚠️ 良好：
  - 页面加载 < 3000ms
  - TTFB < 1000ms
  - LCP < 4000ms

❌ 差：超过上述数值
```

#### 🔍 阶段性能分析

```
DNS 查询：< 100ms 优秀，> 200ms 警告
建立连接：< 100ms 优秀，> 300ms 警告
网络延迟：< 500ms 优秀，> 1000ms 警告
DOM 解析：< 500ms 优秀，> 1000ms 警告
```

**性能瓶颈识别**：
```
如果某个阶段柱子特别长，则该阶段是瓶颈
- 长 DNS -> 检查 DNS 服务配置
- 长 TCP -> 检查 CDN/服务器位置
- 长延迟 -> 优化 API 响应
- 长 DOM -> 检查页面复杂度
```

#### 📦 资源统计

```
总请求数：HTTP 请求总数
总传输数据：所有资源的传输大小

优化建议：
- 请求数 > 100：考虑合并资源
- 传输 > 5MB：检查图片压缩、代码分割
```

#### 💾 内存使用

```
已用内存：当前 JS 堆占用
总内存/限制：浏览器允许的最大内存

警告触发：
⚠️ 超过 80% - 可能内存泄漏
❌ 超过 95% - 立即检查代码
```

#### 🐌 最慢的资源

```
显示加载最慢的前 3 个资源
- 脚本文件
- 样式表
- 图片等

优化方向：
- 代码分割：拆分大脚本
- 异步加载：defer/async 属性
- 按需加载：Lazy Load 图片
```

#### 📈 性能评分

```
0-50 分（差）：立即优化
50-70 分（良好）：继续改进
70-100 分（优秀）：保持水平
```

### 6️⃣ 多租户验证 (`aiops-multitenant-verifier.user.js`)

**功能概述**：
- 快速切换租户进行测试
- 验证数据隔离（同一数据集不应跨租户可见）
- 检测越权访问和数据泄露

**使用方法**：

#### 打开/关闭（Alt+T）
```
按 Alt+T 显示或隐藏多租户验证面板
面板位置：屏幕左下角
```

#### 🔄 租户快速切换

**预定义租户**：
```
1. 全局租户（系统租户）
2. Tenant-A
3. Tenant-B
4. Tenant-C
```

**切换方式**：
```
1. 点击租户卡片
2. 系统保存选择到 localStorage
3. 自动重新加载页面
4. 新租户上下文生效
```

**当前租户信息**：
```
显示：
- 租户名称
- 租户 ID（UUID）
```

#### 🔒 数据隔离检查

**检查范围**（8 个关键 API）：
```
- /api/users - 用户数据
- /api/tenants - 租户配置
- /api/roles - 角色定义
- /api/permissions - 权限数据
- /api/devices - 设备数据
- /api/stations - 站点数据
- /api/orders - 订单数据
- /api/settlement - 结算数据
```

**检查内容**：
```
✅ 隔离（Pass）：
  数据包含 tenant_id 字段
  所有数据都属于当前租户

⚠️ 需检查（Warning）：
  缺少 tenant_id 字段
  包含其他租户的数据

❌ 失败（Fail）：
  API 请求错误
  无法获取数据
```

#### 🧪 自动化测试

**检测数据泄露**：
```
1. 在 Tenant-A 中记录所有用户 ID
2. 切换到 Tenant-B
3. 检查 Tenant-B 的用户列表
4. 如果包含 Tenant-A 的用户 -> 数据泄露！
```

**检测权限越界**：
```
测试受限路径：
- /api/admin/permissions
- /api/admin/role-users
- /api/audit-logs
- /api/system/config

预期结果：普通用户应该得到 403 Forbidden
如果返回 200 -> 权限验证失败！
```

#### 📊 测试结果

```
数据格式：
{
  "check": "用户数据隔离",
  "status": "pass" | "fail" | "warning",
  "message": "✅ 数据隔离正确"
}
```

```
❌ 危险：已用内存 > 95% 上限

处理建议：
1. 检查是否有循环引用
2. 及时清除事件监听器
3. 释放大对象引用
```

```
连接状态：
✅ 在线：设备已连接到网络
❌ 离线：设备离线

连接类型：

3g：中速网络
2g：低速网络
unknown：无法检测
```

## 故障排查

### 问题 1：脚本无法加载

**症状**：按快捷键无反应，脚本面板不显示

**解决方案**：
```
1. 检查 Tampermonkey 是否已启用
   - 点击浏览器工具栏 Tampermonkey 图标
   - 确保状态为启用（蓝色）

2. 检查脚本是否已安装
   - 打开 Tampermonkey 仪表板 (Ctrl+Shift+E)
   - 查看脚本列表中是否包含该脚本
   - 确保脚本状态为启用（复选框打勾）

3. 检查网址匹配
   - 脚本只在 localhost 生效
   - 确保你访问的网址符合条件

4. 强制重新加载页面（Ctrl+Shift+R）

5. 检查浏览器控制台是否有错误
   - F12 打开开发者工具
   - 查看 Console 标签
```

### 问题 2：表单未能正确填充

**症状**：按 Alt+F 后，表单字段仍为空

**解决方案**：
```
1. 检查页面类型检测
   - F12 打开控制台
   - 输入：console.log(document.location.pathname)
   - 检查当前页面 URL

2. 检查表单字段选择器
   - 脚本可能未识别该表单类型
   - 尝试手动编辑 CONFIG 对象中的页面检测规则

3. 检查是否是 React 组件
   - React 表单需要特殊处理
   - 脚本有 nativeInputValueSetter 处理
   - 如仍不工作，请反馈

4. 尝试手动填充
   - 右键点击输入框 -> "自动填充"
```

### 问题 3：API 调试器监控不到请求

**症状**：API 监控面板为空，看不到请求日志

**解决方案**：
```
1. 检查是否已打开面板
   - 按 Alt+A 打开 API 监控

2. 检查请求方式
   - 脚本监控 Fetch 和 XMLHttpRequest
   - 如果页面使用其他库（如 axios），需要额外配置

3. 尝试执行操作
   - 刚打开面板时可能没有请求
   - 执行某个操作（搜索、创建等）来触发 API 调用

4. 检查跨域请求
   - 如果 API 来自不同域，可能被 CORS 阻止
   - 查看浏览器控制台是否有 CORS 错误

5. 刷新页面
   - 脚本在页面加载时安装拦截器
   - 刷新页面后重试
```

### 问题 4：权限测试器无法切换角色

**症状**：点击角色卡片没有反应

**解决方案**：
```
1. 检查 localStorage 权限
   - 脚本需要保存角色到 localStorage
   - 确保浏览器未禁用 localStorage

2. 检查网站隐私模式
   - 隐私模式下 localStorage 不可用
   - 使用普通模式访问

3. 手动切换角色（备选方案）
   - F12 打开控制台
   - 输入：localStorage.setItem('currentTenantId', '角色ID')
   - 刷新页面

4. 检查后端支持
   - 确保后端支持通过 localStorage 传递租户/角色信息
   - 检查 API 是否正确读取该值
```

### 问题 5：性能分析器显示数据不完整

**症状**：某些性能指标为 undefined 或 N/A

**解决方案**：
```
1. 这是正常现象
   - LCP（Largest Contentful Paint）只在页面完全加载后才可用
   - 某些指标可能在特定条件下不可用

2. 重新加载页面
   - 新打开的页面可能缺少性能数据
   - 刷新一次后再查看

3. 检查浏览器支持
   - 旧版浏览器可能不支持某些性能 API
   - 升级到最新 Chrome/Firefox 版本

4. 等待数据收集
   - 某些数据需要在用户进行操作后才能收集
   - 等待页面完全加载和交互后再查看
```

### 问题 6：数据操作工具导出失败

**症状**：点击导出按钮后没有文件下载

**解决方案**：
```
1. 检查浏览器下载设置
   - 确保浏览器允许下载
   - 检查下载文件夹权限

2. 检查弹出窗口阻止
   - 某些组件（如 Blob 创建）可能被阻止
   - 检查浏览器是否阻止了本操作

3. 检查表格数据
   - 表格为空的话无法导出
   - 先创建一些数据再导出

4. 尝试不同格式
   - 先尝试导出为 CSV
   - 如果 CSV 失败，问题可能在浏览器端

5. 使用浏览器控制台手动导出
   - F12 打开控制台
   - 查看 window.__aiops_exportData 对象
```

### 问题 7：日志聚合太多日志导致卡顿

**症状**：面板加载缓慢，滚动不流畅

**解决方案**：
```
1. 清空日志
   - 点击"清空"按钮
   - 删除所有已收集日志

2. 过滤日志
   - 取消勾选不需要的日志类型
   - 仅显示需要的日志等级

3. 导出并关闭
   - 点击"导出"保存当前日志
   - 点击"清空"删除日志
   - 关闭面板节省资源

4. 检查内存
   - 打开系统监控（Alt+M）
   - 如果内存占用过高，关闭其他面板
```

### 问题 8：系统监控显示内存泄漏警告

**症状**：看到"⚠️ 内存使用超过 80%"警告

**解决方案**：
```
正常排查步骤：

1. 检查是否持续增长
   - 监控内存 1 分钟
   - 如果持续增长 -> 可能泄漏
   - 如果稳定 -> 正常

2. 找出泄漏源
   - 关闭其他标签页
   - 关闭浏览器扩展（除 Tampermonkey）
   - 重新加载页面

3. 使用 Chrome DevTools 分析
   - F12 -> Memory -> Heap Snapshot
   - 拍摄两张快照（间隔 1 分钟）
   - 对比两张快照找出变化

4. 常见泄漏原因
   - 事件监听器未移除
   - 定时器未清理（setInterval)
   - 大对象引用未释放
   - DOM 节点引用未释放

5. 联系开发团队
   - 提供内存快照
   - 说明重现步骤
```
