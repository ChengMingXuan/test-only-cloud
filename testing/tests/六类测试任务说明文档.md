## 概述

AIOPS 测试体系采用 **六类测试框架** 互补覆盖，从单元测试到性能压测形成完整闭环：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           AIOPS 六类测试体系                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ① pytest ──→ ② Cypress ──→ ③ Playwright ──→ ④ Selenium ──→ ⑤ Puppeteer ──→ ⑥ K6  │
│    API测试      组件测试      E2E测试        兼容性测试     性能监控       压力测试   │
│    (后端)       (前端)        (跨浏览器)     (多浏览器)     (Chrome深度)   (API负载)  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 一、pytest - API单元测试

> **定位**：后端API接口、数据库操作、业务逻辑验证  
> **覆盖率**：100% API端点 | **用例数**：93+ | **耗时**：10-15分钟  
> **路径**：`tests/test-automation/`

### 任务列表

| 任务名称 | 说明 | 核心命令 | 报告 |
|----------|------|----------|------|
| `🧪 pytest：全量测试` | 运行全部93个用例 | `pytest tests/ -v --html=report.html` | `report.html` |
| `🧪 pytest：快速冒烟测试` | 仅 `@smoke` 标记用例 | `pytest tests/ -v -m smoke` | `report-smoke.html` |
| `🧪 pytest：P0优先级` | 认证/权限/租户隔离 | `pytest tests/ -v -m p0` | `report-p0.html` |
| `🧪 pytest：P1优先级` | 充电/结算/设备核心 | `pytest tests/ -v -m p1` | `report-p1.html` |
| `🧪 pytest：CRUD测试` | 增删改查完整性 | `pytest tests/ -v -m crud` | `report-crud.html` |
| `🧪 pytest：多租户隔离测试` | 租户数据隔离验证 | `pytest tests/ -v -m tenant_isolation` | `report-tenant.html` |
| `🧪 pytest：安全测试` | 鉴权/注入/越权 | `pytest tests/ -v -m security` | `report-security.html` |
| `🧪 pytest：生成覆盖率报告` | 代码覆盖率统计 | `pytest --cov=tests --cov-report=html` | `coverage_report/` |

### 标记说明

| 标记 | 含义 | 预计耗时 |
|------|------|----------|
| `@smoke` | 冒烟测试，核心链路快速验证 | 2-3分钟 |
| `@p0` | P0优先级，认证/权限/租户 | 3-5分钟 |
| `@p1` | P1优先级，充电/结算/设备 | 5-8分钟 |
| `@crud` | CRUD增删改查完整性 | 5-8分钟 |
| `@tenant_isolation` | 多租户隔离安全测试 | 3-5分钟 |
| `@security` | 安全测试 | 3-5分钟 |

### 命令示例

```powershell
# 进入测试目录
Set-Location tests/test-automation

# 激活虚拟环境
.\venv\Scripts\Activate.ps1

# 运行全量测试
pytest tests/ -v --html=report.html --self-contained-html

# 运行指定标记
pytest tests/ -v -m "smoke and not slow"

# 运行指定文件
pytest tests/test_auth.py -v
```

```powershell
# 运行多浏览器测试
pytest tests/test_cross_browser.py -v --html=report.html

docker-compose -f selenium-grid-config.yml down
```

| 任务名称 | 说明 | 核心命令 | 输出 |

|------|------|------|--------|

## 七、全量测试 - 分层执行

> **脚本路径**：`scripts/run-all-tests.ps1`  
> 按执行级别分层，从快到全自由选择

### 执行级别

| 任务名称 | 级别 | 耗时 | 包含测试 |
|----------|------|------|----------|
| `🧪 全量：冒烟测试` | smoke | ~5分钟 | ① pytest冒烟 + ⑥ K6冒烟 |
| `🧪 全量：快速验证` | quick | ~20分钟 | ① pytest全量 + ② Cypress |
| `🧪 全量：标准验证` | standard | ~1.5小时 | ① pytest + ② Cypress + ③ Playwright(Chromium) + ⑥ K6负载 |
| `🧪 全量：完整验证` | full | ~3-4小时 | 6类测试代表场景 |
| `🧪 全量：终极验证` | ultimate | ~5-6小时 | 6类测试所有场景 |

### 执行顺序

```
Level: smoke
├── ① pytest 冒烟测试
└── ⑥ K6 冒烟测试

Level: quick
├── ① pytest 全量测试
└── ② Cypress 组件测试

Level: standard
├── ① pytest 全量测试
├── ② Cypress 组件测试
├── ③ Playwright (仅Chromium)
└── ⑥ K6 负载测试

Level: full
├── ① pytest 全量测试
├── ② Cypress 组件测试
├── ③ Playwright (Chromium + Firefox + WebKit)
├── ④ Selenium 多浏览器矩阵
├── ⑤ Puppeteer 性能基准
└── ⑥ K6 压力测试

Level: ultimate
├── ① pytest (全量 + P0 + P1 + 多租户 + 安全)
├── ② Cypress 组件测试
├── ③ Playwright (跨浏览器 + 移动端)
├── ④ Selenium 完整矩阵
├── ⑤ Puppeteer (性能 + 视觉回归)
└── ⑥ K6 (综合 + 充电 + 数字孪生 + 区块链)
```

### 命令行参数

```powershell
# 基本用法
.\scripts\run-all-tests.ps1 -Level <级别>

# 参数说明
-Level        执行级别: smoke | quick | standard | full | ultimate
-StopOnFailure 遇到失败立即停止
-SkipInstall   跳过依赖安装检查

# 示例
.\scripts\run-all-tests.ps1 -Level smoke                      # 冒烟测试
.\scripts\run-all-tests.ps1 -Level full                       # 完整验证
.\scripts\run-all-tests.ps1 -Level ultimate -StopOnFailure    # 终极验证，失败即停
.\scripts\run-all-tests.ps1 -Level full -SkipInstall          # 跳过依赖安装
```

2. 输入 `Tasks: Run Task`
3. 选择目标测试任务
4. 在终端面板查看执行结果

### 方式二：终端命令行

```powershell
# 进入项目根目录
cd D:\2026\aiops.v2

# 运行全量测试脚本
.\scripts\run-all-tests.ps1 -Level standard

# 或直接运行某个测试
Set-Location tests/test-automation
.\venv\Scripts\Activate.ps1
pytest tests/ -v -m smoke
```

## 常见问题

### Q: pytest 测试失败怎么办？
A: 
1. 检查API服务是否启动（`docker ps`）
2. 确认数据库连接正常
3. 查看 `report.html` 中的详细错误信息

### Q: Selenium Grid 启动失败？
A: 
1. 确认Docker已运行
2. 检查端口4444是否被占用：`netstat -ano | findstr 4444`
3. 重启Docker后重试

### Q: K6 提示未安装？
A: 
1. 运行 `choco install k6`
2. 或从 https://k6.io 下载安装包
3. 确认 k6 在 PATH 中：`k6 version`

### Q: Playwright 浏览器下载失败？
A: 
1. 检查网络连接
2. 运行 `npx playwright install --with-deps`
3. 如有代理，配置 `HTTPS_PROXY` 环境变量

### Q: Cypress 打开后白屏？
A: 
1. 删除 `node_modules` 重新安装
2. 检查 Node.js 版本 >= 18
3. 运行 `npx cypress verify`
